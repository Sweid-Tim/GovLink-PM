# Claudian 团队共享 iCloud 修复方案

## 问题根因

GovLink 的 Obsidian vault 放在 iCloud 共享目录中，Claudian 插件把个人本机状态也写进了共享目录：

- `.obsidian/plugins/claudian/data.json`：保存当前打开的 Claudian 标签页和 conversationId
- `.claudian/sessions/*.meta.json`：保存 Claude Code sessionId
- `.claude/sessions`：旧版本可能使用的会话目录

Claude Code 的 sessionId 只能在创建它的机器/会话环境里恢复。团队成员共享这些文件后，A 同事生成的 sessionId 会同步给 B 同事，B 同事再打开 Claudian 时就会尝试恢复 A 的旧会话，于是报：

```text
No conversation found with session ID: ...
```

## 最终修复策略

把 Claudian 的个人状态按电脑名隔离：

- 共享知识库内容继续放在 iCloud
- Claudian 会话改为 `.claudian/<电脑名>/sessions`
- Claudian 设置改为 `.claudian/<电脑名>/claudian-settings.json`
- Claudian 打开的标签页状态改为 `data.json -> tabManagerStateByHost -> <电脑名>`
- 遗留的共享 `.claudian/sessions` 和 `.claude/sessions` 自动改名禁用
- 历史会话不再把旧 Claude Code `sessionId` 作为 resume 参数传回 Claude Code，避免旧会话失效时反复报错

这样每个同事只会读写自己的 Claudian 会话，不会互相污染。

## 管理员第一次执行

先完全退出 Obsidian，然后运行：

```bash
cd "/Users/tianye/Documents/New project"
./claudian-team-doctor.sh install
```

看到 `Verification passed.` 就说明修复完成。

## 每位同事执行

同事也需要在自己的电脑上运行同一个脚本。若他们的 vault 路径不同，把路径放到命令最后：

```bash
cd "/Users/tianye/Documents/New project"
./claudian-team-doctor.sh install "/他们电脑上的/GovLink-PM/路径"
```

运行前必须完全退出 Obsidian。

## 日常自检

任何人如果遇到 Claudian 报错，先运行：

```bash
cd "/Users/tianye/Documents/New project"
./claudian-team-doctor.sh verify
```

健康状态会显示：

```text
Verification passed.
This computer uses isolated Claudian storage: .claudian/<电脑名>
```

## 插件升级后

Claudian 插件升级可能覆盖 `main.js`，导致隔离补丁失效。升级后重新运行：

```bash
cd "/Users/tianye/Documents/New project"
./claudian-team-doctor.sh install
```

## 紧急清理

如果只是要清掉共享状态，不重新补插件：

```bash
cd "/Users/tianye/Documents/New project"
./claudian-team-doctor.sh sanitize
```

## 关键纪律

- 不要在 Obsidian 打开时运行修复脚本
- 不要手动恢复 `.claudian/sessions` 或 `.claude/sessions`
- 不要从别人电脑复制 `.claudian/<电脑名>` 目录
- 插件升级后必须重新 `install`
