## 我们项目部现在面临的挑战

### 你每天都在做的事

| 工作   | 现状                     | 花费时间       |
| ---- | ---------------------- | ---------- |
| 酒店方案 | 打开携程、Booking、维途...一个个查 | 1-2 小时/次   |
| 航班方案 | 查航班、比价、确认三公标准          | 1-2 小时/次   |

| 制作标书 | 翻模板、凑内容、改格式            | N 小时/次     |

### 现在的痛点

```
😫 每次做方案都要从零开始查资料
😫 好不容易找到的好酒店，做完项目就忘了
😫 别人踩过的坑，我再做一遍
😫 三公经费标准总是记不住
😫 制作标书要从零开始凑内容，改格式改到崩溃
😫 新人上手要靠师傅带，师傅忙就搁置
```

### 我们想要改变什么

> **让 AI 帮你做那些"查资料、填表格、比价格"的重复工作**
> **你只需要做判断和决策**

## 什么是 AI 原生工作流

### 一个更贴切的比喻：你的超级助理

想象你有一个超级能干的助理——她 24 小时待命，不用休息，从不抱怨，而且记忆力超好。

```
👩‍💼 以前（你干活）：
   你 → 打开携程 → 一家家查酒店评分
   你 → 打开Booking → 再查一遍评价
   你 → 打开维途 → 比价格
   你 → 翻历史文档 → 找之前的利雅得案例
   你 → 查三公标准PDF
   你 → 填16字段Excel表格
   ⏰ 1-2小时后：方案还没弄完...

👩‍💼 现在（AI干活）：
   你 → 打开Claudian → 输入："帮我制作6月中旬沙特和阿联酋的酒店方案"
   AI → 5分钟后，方案初稿已生成
   你 → 审核、调整、发出
   ⏰ 5分钟搞定！
```

### 对项目部来说，这意味着什么

| 以前             | 现在                |
| -------------- | ----------------- |
| 你一个个网站查酒店      | AI 帮你查好、对比好、推荐最优解 |
| 你翻历史记录找类似案例    | AI 直接告诉你"上次做过类似的" |
| 你手动填 16 字段酒店表格 | AI 生成初稿，你审核       |
| 踩过的坑下次又踩       | AI 帮你记住，下次自动规避    |
| 你从零开始写标书       | AI 从历史标书提取内容，生成初稿 |

### 核心转变

```
传统模式：你干活，工具只是工具
AI模式：AI干活，你做判断（审核、决策、沟通）
```

## 我们的工具

### 工具一：Obsidian（知识库）

**可以理解为：你的"团队知识共享笔记本"**

```
普通笔记：你写完就躺在那里，下次还得自己翻
Obsidian：AI 能读懂、帮你搜、帮你整理、帮你关联
```

**它就像一个超级笔记本，能记住所有事情**：
- 📍 酒店库（哪家好、哪家踩过坑，👩‍💼 你的助理都记得）
- 📍 供应商库（哪个地接靠谱、哪个导游专业）
- 📍 历史方案（上次做的日本团组是怎么安排的）
- 📍 三公经费标准（利雅得的住宿标准是多少）
- 📍 SOP 操作规范（报批流程、16字段表格怎么填）
- 📍 历史标书（上次投的标是怎么写的）

### 工具二：Claude Code（AI 执行者）

**可以理解为：你的"超级助理"**

```
ChatGPT：像 Siri，你问一句它答一句（被动响应）
Claude Code：像能干的助理，你交代任务它帮你完成（主动执行）
```

**你的助理能帮你做什么**：
- 📝 生成酒店方案初稿（不用自己一个个网站查）
- 📝 生成航班方案（比价、三公核查一起做）
- 📝 制作标书（从历史标书提取内容，30分钟出初稿）
- 📝 查找历史案例（"上次那个日本团是怎么做的来着？"）
- 📝 整理会议记录（自动提取行动项）
- 📝 沉淀项目经验（做完就忘？不存在的）

### 工具三：Claudian 插件

**就是让助理直接在你的笔记本里工作**

```
没有 Claudian：
你想用 AI → 关掉 Obsidian → 打开 ChatGPT → 复制粘贴 → 再贴回来

有 Claudian：
你在 Obsidian 里 → 直接跟助理说话 → 助理帮你做好 → 直接写在笔记本里
```

**就像助理就坐在你旁边，不用跑来跑去**

**效果对比**：

## 记住这三点

### 1. AI 是助手，不是替代者

```
AI 的工作：查资料、填表格、比价格、出初稿
你的工作：判断对不对、决定用哪个、和客户、供应商沟通
```

### 2. AI 生成的要审核

```
AI 可能的问题：
- 信息可能过时（价格、评分）
- 理解可能有偏差
- 不了解最新的变化

你的价值：判断 AI 说的对不对
```

### 3. 从简单开始

```
先从这些开始：
✅ "帮我查一下三公经费"
✅ "帮我找一个历史案例"
✅ "帮我写一封邮件"

然后再试：
→ "帮我生成完整的酒店方案"
→ "帮我生成完整的标书初稿"
```

## 注意事项

### AI 生成的要审核

```
AI 可能的问题：
- 信息可能过时（价格、评分）
- 理解可能有偏差
- 不了解最新变化

你的价值：判断对不对
```

**审核要点**：
- 关键数据对不对？（价格、评分）
- 方案是否符合客户需求？
- 有没有遗漏重要信息？

### 学会追问

如果 AI 第一次回答不够好：

```
"再详细一点"
"换个角度考虑"
"加入 [某个特殊要求]"
"排除 [某个选项]"
```

### 从简单开始

```
先从这些开始：
✅ "帮我查 [城市] 的三公经费"
✅ "帮我找 [国家] 的历史案例"
✅ "帮我查 [酒店名] 的评价"

然后再试：
→ "帮我生成完整的酒店方案"
→ "帮我生成完整的标书初稿"
→ "帮我生成完整的行程框架"
```

## 常见问题

### Q1：AI 说的信息准确吗？

**不一定 100% 准确**。需要你审核：
- 价格可能有时效性
- 评分可能变化
- 特殊要求可能没考虑到

### Q2：我不知道该问什么？

从这几个简单句式开始：
- "查一下..."
- "帮我找..."
- "告诉/记录..."

### Q3：AI 理解错了怎么办？

继续追问，给更多背景信息：
```
第一次："帮我做酒店方案"
AI 可能理解错了

第二次："是政府团组，8人，要住五星级，预算有限"
AI 理解更准确了
```

### Q4：我不确定 AI 找的信息对不对？

可以这样验证：
- 让 AI 指出信息来源
- 自己去对应笔记核实
