# GovLink项目管理知识体系

> 目标：把大美政通正在做的出境公务团组、涉外公务资源开发、销售市场项目、投标项目、入境游与入境接待业务，统一整理到项目管理知识体系下，形成一个可被 LLM 和团队成员反复调用的项目管理技能。

## 一句话定位

GovLink 的项目管理不是抽象的管理学，而是把“客户需求、方案范围、倒排计划、资源协同、质量交付、风险预警、合规边界、复盘沉淀”串成一个闭环。

以后凡是生成方案、排计划表、拆任务、做执行跟进、准备客户交付物、评估风险、复盘项目，都应先读取本页和 [[wiki/summaries/SOP/17-项目管理助手]]。

## 适用业务场景

| 业务场景 | 项目管理重点 | 典型产出 |
|:---|:---|:---|
| 出境公务团组 | 范围、进度、成本、质量、风险、干系人 | 报批行程、实际行程、倒推表、预算表、出团手册、复盘 |
| 境外公务资源开发 | 范围、沟通、采购/外部协作、风险、干系人 | 公务需求单、客户画像、资源推荐表、外发口径、跟进记录 |
| 销售与市场项目 | 整合、范围、进度、沟通、干系人 | 客户开发计划、市场调研报告、线索跟进表、拜访材料 |
| 投标项目 | 整合、范围、进度、成本、质量、风险、采购、干系人 | 投标可行性评估、标书任务拆解、复审清单、投标文件 |
| 入境游/入境接待 | 范围、进度、质量、资源、采购、风险、沟通 | 接待方案、用车酒店餐厅安排、接待手册、报价与结算 |
| APEC/签证/辅助业务 | 范围、进度、质量、沟通、风险 | 资质自查表、材料清单、进度跟进表、交付说明 |

## 十大知识领域与 GovLink 映射

### 1. 项目整合管理

**核心问题**：这个项目为什么做？谁拍板？哪些工作必须被放进同一个闭环？

在 GovLink 中，整合管理负责把销售、项目、市场、公务、财务、供应商和客户需求拉到同一张图里。

| GovLink动作 | 管理要求 | 输出物 |
|:---|:---|:---|
| 项目启动 | 明确客户、目标、成功标准、负责人 | 项目启动卡 |
| 方案统筹 | 统一行程、预算、公务、航酒签证、报批节奏 | 项目总控表 |
| 变更处理 | 记录客户改动、航班涨价、资源变动、人员变动 | 变更记录表 |
| 复盘归档 | 把经验沉淀回 wiki 和历史数据库 | 复盘页、资源更新记录 |

**GovLink判断标准**：如果一个项目同时牵涉客户、境外资源、供应商、费用、报批和交付，就必须有一个总控入口，而不是散落在聊天记录里。

### 2. 项目范围管理

**核心问题**：我们到底交付什么？哪些不在本次服务范围内？

范围管理尤其适用于方案报价、投标文件、入境接待和公务资源开发。范围不清会导致客户期待失控、供应商口径失控、内部重复劳动。

| 场景 | 范围边界示例 |
|:---|:---|
| 出境团组 | 是否包含航班、酒店、签证、公务、餐厅、导游、保险、邀请函、出团手册 |
| 境外公务 | 推荐几家单位、是否负责正式预约、是否代写邀请函、是否负责现场陪同 |
| 入境接待 | 是否包含机场接送、翻译、会务、用车、酒店、餐饮、城市参观、付款代垫 |
| 销售市场 | 是市场调研、线索开发、客户拜访，还是已进入项目转化 |
| 投标 | 商务标、技术标、报价标、盖章装订、现场开标分别由谁负责 |

**必须输出**：范围说明 + 不包含事项 + 客户需提供资料 + 变更触发条件。

### 3. 项目进度管理

**核心问题**：每一步什么时候完成？哪些事项会卡住后面的动作？

GovLink 的进度管理应采用“出发日/投标截止日/接待日倒推”的方式。

| 项目类型 | 倒排基准 | 关键节点 |
|:---|:---|:---|
| 出境团组 | 出发日 D-day | 报批、护照签证、航班出票、酒店确认、公务确认、行前会 |
| 投标项目 | 投标截止时间 | 购标、共读、初稿、资料补齐、复审、盖章、上传/递交 |
| 境外公务 | 客户要求出方案日期/可公务日期 | 需求解析、国家调研、资源筛选、外发、跟进、推荐 |
| 入境接待 | 抵达日期 | 需求确认、报价、资源锁定、付款、接待手册、现场执行 |
| 市场销售 | 拜访/转化目标日期 | 客户名单、资料准备、触达、跟进、复盘、下一步 |

**关键原则**：进度表不是任务清单，而是风险预警工具。任何未完成事项都要标注影响对象和最晚决策时间。

### 4. 项目成本管理

**核心问题**：预算是否合规？报价是否覆盖成本和风险？现金流是否安全？

| 成本类型 | 管理重点 |
|:---|:---|
| 三公经费 | 住宿、伙食、公杂优先查 2017 标准，缺失再回退 2013 标准 |
| 航班酒店 | 价格波动、退改政策、预订时限、是否需政采 |
| 地接导游 | 报价口径、服务边界、付款节点、罚则合理性 |
| 入境接待 | 车辆、翻译、会务、酒店、餐饮、门票、物料、税费 |
| 投标 | 标书费、保证金、印刷装订、授权资料、现场差旅 |

**必须输出**：成本明细 + 报价假设 + 不确定费用 + 付款/垫付风险。

### 5. 项目质量管理

**核心问题**：交付物是否能被客户直接使用？现场服务是否稳定？

GovLink 的质量不只是“文字漂亮”，还包括真实、合规、可执行、可复核。

| 交付物 | 质量标准 |
|:---|:---|
| 航班方案 | 航班真实、时间合理、行李/中转/价格清楚、符合出入境节奏 |
| 酒店方案 | 评分、面积、早餐、取消政策、位置、三公经费均可核查 |
| 行程方案 | 报批版避开禁用词，实际版全过程模拟，公务安排在工作日 |
| 公务推荐 | 单位真实、联系方式真实、与客户需求匹配，不虚构资源 |
| 投标文件 | 逐条响应招标文件，星号条款、签字盖章、格式、份数无遗漏 |
| 入境接待 | 路线、车辆、餐饮、会务、翻译、礼宾衔接无断点 |

**质量门槛**：凡是要外发或交付客户的内容，必须经过事实核查、格式核查、合规核查、脱敏核查。

### 6. 项目资源管理

**核心问题**：谁做？靠哪些内部和外部资源完成？

| 资源类型 | GovLink例子 | 管理方式 |
|:---|:---|:---|
| 内部角色 | 销售、项目专员、市场、公务开发、财务、管理层 | RACI 分工表 |
| 外部供应商 | 地接、导游、酒店、车队、票代、翻译、会务公司 | 供应商确认表 |
| 境外机构 | 政府、商协会、企业、高校、律所、咨询机构 | 公务资源跟进表 |
| 知识资源 | SOP、历史酒店/餐厅/导游、公务资源、复盘 | wiki 引用 |

**原则**：复杂项目必须建立共享文档或项目总控表，避免信息只存在于个人聊天记录。

### 7. 项目沟通管理

**核心问题**：谁需要知道什么？什么时候同步？用什么口径？

GovLink 的沟通管理有两个层面：内部协作透明、外部口径克制。

| 对象 | 沟通重点 | 注意事项 |
|:---|:---|:---|
| 客户 | 方案进度、待确认事项、风险提示、决策点 | 不甩锅，给选项和建议 |
| 销售 | 客户背景、商务判断、下一步推进 | 不能只同步执行细节 |
| 项目团队 | 任务、截止时间、依赖、阻塞项 | 复杂项目用共享文档 |
| 供应商 | 服务需求、时间人数、标准、报价 | 客户信息默认脱敏 |
| 境外机构 | 交流目的、团组画像、合作价值 | 不泄露敏感客户全称 |

**外发红线**：前期询资源、询接待、询价、供应商沟通必须客户脱敏，末尾标注：`客户名称已脱敏，正式披露前需由项目负责人确认。`

### 8. 项目风险管理

**核心问题**：什么可能出问题？概率多大？影响多大？提前怎么处理？

风险管理要覆盖时间、成本、质量、合规、安全、政治环境、战争冲突、公共卫生、签证、供应商、客户决策等维度。

#### 风险分类

| 风险类别 | 典型风险 |
|:---|:---|
| 进度风险 | 护照/签证延误、公务单位迟迟不确认、客户审批慢、投标截止紧 |
| 成本风险 | 航班涨价、酒店满房、汇率波动、供应商临时加价 |
| 质量风险 | 公务匹配度低、导游服务不稳定、行程过密、材料错误 |
| 范围风险 | 客户不断新增需求、服务边界不清、投标响应遗漏 |
| 合规风险 | 三公经费超标、报批行程用词不当、客户信息外泄 |
| 供应商风险 | 地接不靠谱、车辆/导游临时变更、酒店政策变化 |
| 外部环境风险 | 政治动荡、战争冲突、罢工、自然灾害、大型活动、节假日 |
| 安全风险 | 治安、交通、公共卫生、极端天气、团员身体状况 |
| 沟通风险 | 客户真实需求未被理解、销售与项目口径不一致、境外回复慢 |

#### 风险登记册模板

| 编号 | 风险描述 | 类别 | 概率 | 影响 | 等级 | 触发信号 | 应对策略 | 负责人 | 截止时间 | 状态 |
|:---:|:---|:---|:---:|:---:|:---:|:---|:---|:---|:---:|:---:|
| R1 | 目标城市同期大型展会导致酒店涨价或满房 | 成本/进度 | 中 | 高 | 高 | 酒店报价高于历史均值、可订房少 | 提前锁房，准备备选城市/区域 | 项目专员 | D-45 | 跟进中 |
| R2 | 境外公务单位迟迟不确认 | 进度/质量 | 高 | 高 | 高 | 邮件 3 个工作日无回复 | 同步开发 3-5 家备选单位，升级电话/LinkedIn | 公务开发 | D-30 | 跟进中 |
| R3 | 报批行程出现敏感或禁用表达 | 合规/质量 | 中 | 高 | 高 | 文案出现“考察、参访、景点、参观游览” | 按报批行程 SOP 复审替换 | 项目专员 | 提交前 | 未开始 |

#### 风险评分规则

| 概率/影响 | 低 | 中 | 高 |
|:---|:---|:---|:---|
| 概率 | 小概率，通常可控 | 有明显可能，需要跟踪 | 很可能发生或已有征兆 |
| 影响 | 可内部消化 | 影响客户体验或项目成本 | 影响成团、合规、交付或客户关系 |

等级判断：高概率 + 高影响为高风险；中概率 + 高影响通常也按高风险管理。

### 9. 项目采购管理

**核心问题**：哪些东西需要外部采购？供应商是否可靠？合同和付款是否安全？

| 采购对象 | 采购检查点 |
|:---|:---|
| 航班 | 航司、舱位、行李、退改、出票时限、政采要求 |
| 酒店 | 位置、评分、面积、早餐、取消政策、发票/付款方式 |
| 地接/导游/车队 | 报价、服务范围、语言能力、紧急联系人、历史评价 |
| 餐厅 | 团餐标准、位置、菜单、宗教/饮食禁忌、付款方式 |
| 会务/翻译 | 专业领域、语言、现场流程、设备、发票 |
| 入境接待资源 | 车辆、酒店、餐厅、活动场地、讲解/陪同 |

**原则**：采购不是找最低价，而是找“可交付、可追责、可替代”的资源。

### 10. 项目干系人管理

**核心问题**：谁会影响项目成败？他们关心什么？如何管理期待？

| 干系人 | 关注点 | 管理方式 |
|:---|:---|:---|
| 政府/事业单位客户 | 合规、级别、成果、费用、报批 | 正式、稳妥、可追溯 |
| 企业客户 | 效率、性价比、商业机会、保密 | 明确 ROI 与时间安排 |
| 销售伙伴 | 客户关系、成交机会、报价策略 | 早同步、给判断依据 |
| 项目伙伴 | 可执行性、资源、时间、风险 | 任务清晰、节点明确 |
| 境外机构 | 合作价值、对等性、时间成本 | 邮件专业、诉求明确 |
| 供应商 | 服务边界、付款、责任 | 需求清晰、客户脱敏 |
| 财务/法务/管理层 | 合规、合同、收款、风险 | 关键节点提前介入 |

## 项目生命周期

| 阶段 | 核心问题 | 必读知识领域 | 关键产出 |
|:---|:---|:---|:---|
| 启动 | 是否值得做？目标是什么？ | 整合、干系人、范围、风险 | 项目启动卡、客户画像 |
| 规划 | 怎么做？谁做？何时交付？ | 范围、进度、成本、资源、沟通、风险 | 项目计划表、预算、风险登记册 |
| 执行 | 按计划推进了吗？资源到位了吗？ | 资源、采购、沟通、质量 | 执行跟进表、供应商确认表 |
| 监控 | 哪里偏离了？要不要变更？ | 整合、进度、成本、质量、风险 | 变更记录、风险更新、客户同步 |
| 收尾 | 交付是否完成？经验是否沉淀？ | 质量、沟通、干系人、整合 | 复盘、归档、wiki 更新 |
